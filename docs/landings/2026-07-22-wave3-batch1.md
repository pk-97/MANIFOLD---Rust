## Deviations from the design (all ruled on the bus)

1. **W3-D1** — P3-R tests-out shape: D3's literal `tests/mod.rs` nesting would rename every test path (fails the LIST gate); as-built uses `#[cfg] #[path = "tests/<f>.rs"] mod <orig>_tests;`. Design doc D3 row updated in this landing.
2. **W3-D2/D3/D4** — the verifier needed three new wiring classes (1-to-1 test-mod conversion; context-cfg variant; use-block opener), all "arm-before-is_moved" fixes for git self-paired lines; shipped as ONE shared commit V2 with fixtures.
3. **D-45 LIST gate caught a real defect:** P3-C's tests split initially flattened `dispatch_contract_tests` → `tests`, renaming 3 test paths. Found by the observed exec-vs-p3g LIST cross-diff (an earlier "derived" pass had missed it — derivation-isn't-observation, confessed and corrected by the seat). Fixed by d4283f9a; LIST diff-empty confirmed by observation.
4. **W3-D5** — the crashed P3-G seat's split was residue-0 but didn't compile (73 missing-wiring errors). Lesson now on the bus: **residue 0 ≠ compiles**; extraction isn't complete until `cargo check` (incl. gpu-proofs config) is green. Completed as wiring commit e6e767e7.
5. **W3-D4(3)** — P3-R's final range carries residue 5, accepted as enumerated `pub(super)` widenings (D-9 eyeball precedent, no verifier allow added): `scale: f32`, `width: u32`, `height: u32`, `effect_id: EffectId`, `effect_type: PresetTypeId` — each proven clean per-commit; the range orphans are git short-line mispairing.
