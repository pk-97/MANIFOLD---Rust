## What shipped

- `ZTier { Base, Chrome, Overlay, Ghost }`, `RegionToken`, `UITree::begin_region(rect, tier, label, extra_flags)` / `end_region`, `ALLOW_OVERFLOW = 1 << 8` — the region mechanism (`manifold-ui/src/tree.rs`, `node.rs`).
- Region list (`regions: Vec<(ZTier, NodeId)>`) with stable tier-sorted traversal in `traverse()`/`traverse_range()`; sort at build, zero per-frame alloc.
- Every main-window panel in `ui_root.rs` built under a region: transport/header/footer → `Chrome`; inspector (both `build` and `build_in_rect`), viewport (both call sites), layer_headers, split/inspector resize handles → `Base`/`Chrome`; overlay driver → `Overlay`; toasts/drag-ghost → `Ghost` (+`ALLOW_OVERFLOW`).
- **D4 enforcement, two arms:** a runtime `debug_assert` in `mint` rejecting a `parent:None` node minted outside an open `begin_region` bracket (app debug builds), plus `all_roots_are_regions()` with tests `all_roots_are_regions_holds_for_a_properly_regioned_tree` and `all_roots_are_regions_catches_a_stray_root` (proves the check catches a violation, not just passes clean).
- **BUG-060 stopgap removed** (Peter's explicit P1 condition): the root-parented `content_clip` ClipRegion in `inspector.rs` is deleted — the ScrollContainers (`master_scroll`/`layer_scroll`) already mint their own `CLIPS_CHILDREN` clip at the same `columns_y`, so it was always a strict subset; the outer inspector region now carries footer containment. Guard test `layer_scroll_clip_prevents_scrolled_columns_painting_over_the_tab_strip` added. The `build_toggle_trigger_row` reveal clip is kept (nested, not root-parented, does per-frame-animated work no static ancestor substitutes for; `trigger_gate_drawer_tween_clips_midflight` still green).
- Gate scenes `bug060`/`bug047` added to `ui_snapshot/fixtures.rs`; L3 flow `scripts/ui-flows/bug060-inspector-footer-containment.json`.

## Deviations from the brief (all reasoned, none silent)

- `begin_region` gained a 4th param `extra_flags` — the design's committed 3-param signature can't
  carry `ALLOW_OVERFLOW` (D3), and the doc's own prose requires it. Doc-internal, no architectural fork.
- **Escalation (design-level, surfaced to Peter):** the design's "renderer walks root children via
  `traverse()`" anchor is wrong for the live main window, which renders via `panel_cache_info()` +
  `UICacheManager` (array-ordered). Worker made `traverse()` tier-sorted (covers snapshots + editor)
  and relied on D1 containment for the cache path rather than reordering it. Accepted for P1 (BUG-060
  dies by containment); D2 tier-order on the cache path carried as **VD-018**, close in P2.
- Editor-window nodes + snapshot scratch trees wrapped defensively (not the P2 editor migration) —
  unwrapped they panic the D4 assertion on first debug render. Scope-minimal, documented in-code.
- Stopgap decision reached via a unit-test clip-stack replica, not a PNG (the brief said "decide by
  PNG") — equivalent rigor, disclosed; the BUG-076 scroll limit made a true-bottom PNG unreachable.
