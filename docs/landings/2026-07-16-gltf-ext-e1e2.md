## Deviations from brief

- **D3's VERIFY-AT-IMPL resolved as "no intermediate exists"** — the orchestrator confirmed `render_scene`'s `second_pass` mechanism runs inside the *same* Metal render pass as the opaque draws (`crates/manifold-gpu/src/metal/encoder.rs:153-159`), with no framebuffer-fetch exposed. Per the doc's own authorization, E2 split into E2a (build the opaque-resolved intermediate seam) and E2b (the actual refraction/tint shading using it) as two separate worker sessions.
- **E2a's depth-resolve approach differed from the orchestrator's own brief assumption.** The brief suggested reusing the existing `depth_resolve` field (built for volumetric shafts) to get a reloadable depth attachment for Pass B. The executing worker found `R32Float` (that field's resolve format) is not a legal Metal depth-*attachment* pixel format, so it built a separate depth-only prepass (reusing the shadow pipeline machinery) instead. Verified by the orchestrator as a correct, in-scope resolution, not an escalation.
- **E2b changed import-side behavior beyond pure shading.** `gltf_import.rs`'s `effective_alpha` formula (`base_color.a * (1 - transmission_factor)`, the old alpha-blend approximation) was replaced with plain `base_color.a` — the "see-through" effect is now carried entirely by the new shader-side refraction/tint, not by alpha compositing. This is a real behavior change to the F-P5/D8 import mapping, documented in `gltf_import.rs`'s comments and the E2b worker's commit message. One existing test's hardcoded alpha expectation was updated to match.
