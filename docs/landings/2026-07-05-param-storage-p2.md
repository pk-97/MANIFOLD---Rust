## Deviations from brief

- **P2 was briefed as "the compile-driven storage swap." It was that, plus three production behavioural gaps that only became observable once the tests compiled — each fixed at the root.** The mechanical test-pass was delegated to four cheap Sonnet agents (one per downstream crate, on disjoint files); running the newly-live tests then exposed three paths that *compiled* but were wrong under the id-keyed model:
  1. **`RevertEffectGraphCommand`** cleared the graph but never removed the orphaned user params from the manifest, so `prune_orphaned_automation` (which keys on `params.get(id).is_none()`) never fired — a reverted graph left its drivers stranded on the live instrument. Fix: revert removes the `UserAdded` params (captured at their positions for undo) *before* the sweep; undo re-inserts them. (`graph.rs`)
  2. **`EditParamMappingCommand`** wrote the reshape only to the `meta.params` save-shadow, but `synth_user_binding` reads the live manifest spec (D6) — so a recalibrated range/curve/invert/label never reached the renderer. Fix: the command mirrors the reshape onto the manifest `Param.spec` (+`calibrated=true`); undo mirrors the reverse. (`commands/effects.rs`)
  3. **`PresetInstance::restore()`** reseeded to the registry template's full param count then zipped a shorter positional `Vec<f32>` snapshot on top, leaving stale tail entries — undoing a generator-type change restored the wrong param count (6 vs 3). Fix: `restore()` truncates the reseeded manifest to the snapshot's arity (the snapshot is authoritative). (`effects.rs`)
  Plus a **fourth cfg-gated path** the default build skips — the `#[cfg(feature="profiling")]` param readout in `content_thread.rs` — was migrated to the manifest.
- **Two pre-existing trunk failures surfaced during the full-workspace sweep** (catalog drift + design-token ratchet), both failing identically on origin/main — evidence a recent UI landing didn't run the full workspace test. Catalog fixed (deterministic regen, labelled chore); design-token left for its lane (BUG-030).
- **`bench_resolve` is a `#[test]`, not a criterion bench** (criterion isn't a dependency), using `std::time::Instant` + min-of-N.
- **128 MB of unreferenced glTF fixtures** (`japanese_apricot`/`lowe` `.glb`) that an early in-worktree `git add -A` had swept onto the branch were removed (`git rm`) — they belong to the glTF lane, are referenced by no test (only `oomurasaki_azalea` is), and don't belong in a param-storage landing. Main's tree is clean. **Followup: the blobs remain in this branch's history; a `git filter-repo --path` purge (or a squash-land) would keep them off origin — deferred as a separate maintenance step so it can't endanger the landing.**

## Shortcuts confessed (rolled up from phase notes)

- **`meta.params` is a consistent save-shadow, not derived-on-save.** User specs ride the wire inline; bundled range edits ride a `calibration` block; `meta.params` is written in lockstep at every append/remove/restore/reshape site so the graph def stays byte-uniform with a bundled preset JSON (preserving P1's byte round-trip). The cost: two structures (live manifest + shadow) must be kept in sync — the `EditParamMapping` gap above was exactly a missed sync site, which is why it's now covered by a round-trip test.
- **The gen-type/paste undo snapshot is still a positional `Vec<f32>`** (a pre-manifest leftover). `restore()`'s truncate honours its arity correctly and the tests pass, but converting those snapshots to carry a `ParamManifest` would remove the positional assumption entirely — a cleaner future step, not required for P2.
- **Funnels kept but id-keyed** (`get_param`/`set_param`/`set_base_param`/…) rather than inlined to `get_mut` everywhere, to keep the touched-latch/base semantics in one place.
- **`set_gen_param_base`/`get_gen_param` keep a positional card-index API** via a transient `manifest.iter().nth(i)` boundary view (never stored) — the card index is a display concern, not identity.

## Click-script for Peter (≤2 minutes)

1. Add a user-exposed param on an effect, hook a driver/LFO to it, then **Revert Graph** on that card — expect: the card returns to the catalog default *and* the driver disappears (it was leaking before the fix). **Undo** — expect: the graph, the param, and the driver all come back.
2. On a user-exposed param, open its mapping and **recalibrate the range** (min/max, or the response curve) — expect: the slider's range and response actually change in the live render (calibration now reaches the renderer via the manifest spec, D6). **Undo** — expect: it snaps back.
3. Change a generator's **type**, then **Undo** — expect: the previous type returns with exactly its original param set (no extra registry-default sliders appended to the tail).
