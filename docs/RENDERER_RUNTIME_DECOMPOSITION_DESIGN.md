Stage translation: `codegen.rs` decides what WGSL reaches the GPU every frame of a show; `preset_runtime.rs` is the live entry point for effect fusion; `gltf_import.rs` is how a model becomes a playable instrument. Nothing in this wave changes a pixel or a millisecond — the value is that the next importer feature is *adding rows to a table*, not threading a 13-parameter call through a 1,300-line function, and that a merge conflict in these files stops being a session-killer during the release push.

**Binding constraints** (DESIGN_AUTHORING §1): *Hot path* — no per-frame body changes; codegen determinism (emitted text = pipeline-cache key, FREEZE_COMPILER_MAP §5) proven by the byte-snapshot gate; table walks in P3-D run at IMPORT time (user-initiated, not per-frame). *Persistence* — none: no type in these files serializes into `.manifold` (v1 review verified; `ImportReport`/`MergePlan`/`ChainError` are runtime-only). BUT the importer's OUTPUT (`EffectGraphDef`) is persisted by projects — P3-D therefore gates on def-level equivalence (INV-R8), not just tests. *Thread residency* — untouched. *Authority* — `FREEZE_COMPILER_MAP.md` stays authoritative; its §2 rows get path/size updates per landing (D5).
