## 2. Decisions

**D1 — Directory modules with a `mod.rs` facade; no new crates; no reshaping.** Each target file becomes a directory module; `mod.rs` re-exports every public item at its existing path, so `commands::graph::X` and `manifold_core::effects::X` stay valid at all ~200 external sites and both `deserialize_with` string paths keep resolving. This is the Wave 1 P-F2a shape (`ui_root/` split) applied twice more. Rejected: *new crates now* (e.g. `manifold-migration`) — Wave 1 D1's rule holds: module seams first, crate promotion is per-seam and evidence-based, later; a crate move would also churn ~200 import sites for zero class-prevention gain. Rejected: *reshaping while moving* (collapsing scene-command boilerplate, deleting suspected sediment) — pure-move and semantic changes never share a commit (Wave 1 D7); improvement candidates go in a notes file for Peter. Rejected: *migrating callers to the new deep paths* — the facade IS the public surface; deep paths are `pub(crate)` implementation layout.

**D2 — The layer vocabulary in this crate pair.** Wave 1's UI layers don't transplant; the model/command matrix is: **Model** (type + its impl + its serde, per domain — owns shape and invariants), **Migration** (load-time normalization/legacy resolution — owns "old files still load"), **Query** (read-only traversal — owns "answer questions without mutating"), **Command** (mutation + undo, per domain — owns "every change is reversible"). Every extracted module maps to exactly one. The register's vocabulary table gains this row at P2-Z. What this buys: a new scene command touches `commands/graph/scene.rs` only; a new legacy-format quirk touches `project/load_migration.rs` only; neither can silently grow the other's file.

**D6 — Pure moves are wire-format-invisible; the substitute oracle rides every landing anyway.** The argument, stated once so nobody re-derives it nervously: serde derive output depends on struct/field/variant **names** and serde attributes only — never on module paths or crate names; the tagged enums here tag by variant name; the two string-path `deserialize_with` hooks are compile-checked paths that resolve through the D1 facade. Corollary: the fusion content key (`serde_json::to_vec(def)`, FREEZE_COMPILER_MAP §8) is equally untouched. Therefore NO load-migration is needed for any move in this design — and any phase that discovers it would need one has left pure-move territory and **parks immediately** (D-29's rule, verbatim: "Any discovered need for load-migration = PARK that phase"). Mandatory gate per landing regardless (D-29): LiveSchool fixture suite green (`cargo test -p manifold-io --test load_project`, main checkout — fixtures are gitignored) PLUS a save→load→save structural self-comparison: load the fixture through the typed loader, save twice through `save_v2_archive`, `project_tool json` both, field-diff empty; quoted in the landing report.

**D7 — Census gates, generalized from Wave 1 — MULTISET, not set.** The Wave-2 equivalent of the 150-variant dispatch census is the **command-type census**, and it must count occurrences: `rg -o 'impl Command for [A-Za-z_0-9]+' crates/manifold-editing/src/commands/ --no-filename | sort | uniq -c` compared as a multiset. A `sort -u` set-census is BLIND here: two distinct types named `RenameGroupCommand` exist (effect_groups.rs:296, graph.rs:3753), so dropping the graph one during its own move slice would leave a set-diff empty (review H1 — the exact Wave 1 census pattern, transplanted naively, fails at exactly one name this wave moves). Baseline captured at P2-G entry, capture-first per D-30 (today's read: 161 raw / 160 unique / 29 in graph.rs). For the core splits the equivalent is the **public-item census**, widened to every item kind that actually occurs in these files (review M5 — `pub const`, `pub mod`, `pub use`, `pub(crate) fn` all exist and an internally-unused `pub const` could vanish silently): `rg -o '^pub(\((crate|super)\))? (struct|enum|trait|fn|const|type|static|mod|use) [A-Za-z_0-9]+' <target> --no-filename | sort | uniq -c`, multiset-compared old file vs new directory. Both are exit-code scripts on the lane gate, not eyeball checks.

**D7a — Verifier extension ships FIRST, as its own tooling commit (review H2).** `move_identity_check.py` today has no ALLOW class for `#[cfg(test)]` attribute lines or brace-form `mod tests {` openers/closers; distributing one flat test mod into ~7 per-module mods multiplies exactly those short header lines, and whether git pairs them against the single removed original is threshold-fragile — the first test-distribution slice would park on phantom residue. Before ANY test-distribution slice (P2-G S7 and both core phases' test slices), one tooling commit adds a narrow test-mod-wiring ALLOW class (`#[cfg(test)]` / `#[cfg(all(test, feature = "…"))]` attribute lines and bare `mod <name> {`/`}` opener-closer pairs on test mods — attribute must be attached to a mod, smuggle-proof: any non-mod line under the class is residue), with two self-test fixtures: test distribution → PROVEN; a code line smuggled into a test-mod header → CAUGHT. D-19's second-lander rule applies verbatim — this script is co-edited by live Wave 1 lanes; whoever lands second merges both pattern sets and reruns the full self-test.

**D8 — Sediment policy: this wave moves, it does not judge.** Unity-port oddities encountered mid-move (dead-looking shapes, duplicated concerns) are recorded in `docs/notes/wave2-sediment-notes.md` with a `git log -S` receipt each, and left byte-identical. One pre-answered instance so nobody "fixes" it: `EffectContainer` looks vestigial and is NOT — `Layer` and `ProjectSettings` implement it (layer.rs:956, settings.rs:433). Deletion candidates become backlog entries or a later semantic phase Peter approves; never riders.

**Consequences, stated honestly:** (1) `git blame` degrades across three more files — mitigated the Wave 1 way: `.git-blame-ignore-revs` entry per pure-move landing, `--color-moved` review. (2) The facade means import paths no longer reveal which submodule owns an item; the cost of zero call-site churn. Accepted: the module tree is one `ls` away. (3) `instance.rs` (~1.5k) and `scene.rs` (~1.8k) remain sizeable — named ceilings, not failures; both are single-concern by the register's own godhood test (size × churn × **mixed concerns**). (4) Splitting tests beside their subjects turns one 4,772-line test mod into ~7 — shared-fixture drift across modules becomes possible; contained by `test_support.rs` being the only home for fixtures (forbidden move: copying a fixture into a module instead of importing it). (5) This wave runs overnight concurrently with Wave 1 P-D (manifold-ui/manifold-app): different crates, but `manifold-core` is upstream of everything — a Wave-2 core landing invalidates the other lane's incremental build. Cost accepted; the build lock (D-14) serializes the heavy gates.
