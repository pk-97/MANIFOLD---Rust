## 2. Decisions

**D1 ‚Äî Directory modules with a `mod.rs` facade; no new crates; no reshaping.** Each target file becomes a directory module; `mod.rs` re-exports every public item at its existing path, so `commands::graph::X` and `manifold_core::effects::X` stay valid at all ~200 external sites and both `deserialize_with` string paths keep resolving. This is the Wave 1 P-F2a shape (`ui_root/` split) applied twice more. Rejected: *new crates now* (e.g. `manifold-migration`) ‚Äî Wave 1 D1's rule holds: module seams first, crate promotion is per-seam and evidence-based, later; a crate move would also churn ~200 import sites for zero class-prevention gain. Rejected: *reshaping while moving* (collapsing scene-command boilerplate, deleting suspected sediment) ‚Äî pure-move and semantic changes never share a commit (Wave 1 D7); improvement candidates go in a notes file for Peter. Rejected: *migrating callers to the new deep paths* ‚Äî the facade IS the public surface; deep paths are `pub(crate)` implementation layout.

**D2 ‚Äî The layer vocabulary in this crate pair.** Wave 1's UI layers don't transplant; the model/command matrix is: **Model** (type + its impl + its serde, per domain ‚Äî owns shape and invariants), **Migration** (load-time normalization/legacy resolution ‚Äî owns "old files still load"), **Query** (read-only traversal ‚Äî owns "answer questions without mutating"), **Command** (mutation + undo, per domain ‚Äî owns "every change is reversible"). Every extracted module maps to exactly one. The register's vocabulary table gains this row at P2-Z. What this buys: a new scene command touches `commands/graph/scene.rs` only; a new legacy-format quirk touches `project/load_migration.rs` only; neither can silently grow the other's file.

**D6 ‚Äî Pure moves are wire-format-invisible; the substitute oracle rides every landing anyway.** The argument, stated once so nobody re-derives it nervously: serde derive output depends on struct/field/variant **names** and serde attributes only ‚Äî never on module paths or crate names; the tagged enums here tag by variant name; the two string-path `deserialize_with` hooks are compile-checked paths that resolve through the D1 facade. Corollary: the fusion content key (`serde_json::to_vec(def)`, FREEZE_COMPILER_MAP ¬ß8) is equally untouched. Therefore NO load-migration is needed for any move in this design ‚Äî and any phase that discovers it would need one has left pure-move territory and **parks immediately** (D-29's rule, verbatim: "Any discovered need for load-migration = PARK that phase"). Mandatory gate per landing regardless (D-29): LiveSchool fixture suite green (`cargo test -p manifold-io --test load_project`, main checkout ‚Äî fixtures are gitignored) PLUS a save‚Üíload‚Üísave structural self-comparison: load the fixture through the typed loader, save twice through `save_v2_archive`, `project_tool json` both, field-diff empty; quoted in the landing report.

**D7 ‚Äî Census gates, generalized from Wave 1 ‚Äî MULTISET, not set.** The Wave-2 equivalent of the 150-variant dispatch census is the **command-type census**, and it must count occurrences: `rg -o 'impl Command for [A-Za-z_0-9]+' crates/manifold-editing/src/commands/ --no-filename | sort | uniq -c` compared as a multiset. A `sort -u` set-census is BLIND here: two distinct types named `RenameGroupCommand` exist (effect_groups.rs:296, graph.rs:3753), so dropping the graph one during its own move slice would leave a set-diff empty (review H1 ‚Äî the exact Wave 1 census pattern, transplanted naively, fails at exactly one name this wave moves). Baseline captured at P2-G entry, capture-first per D-30 (today's read: 161 raw / 160 unique / 29 in graph.rs). For the core splits the equivalent is the **public-item census**, widened to every item kind that actually occurs in these files (review M5 ‚Äî `pub const`, `pub mod`, `pub use`, `pub(crate) fn` all exist and an internally-unused `pub const` could vanish silently): `rg -o '^pub(\((crate|super)\))? (struct|enum|trait|fn|const|type|static|mod|use) [A-Za-z_0-9]+' <target> --no-filename | sort | uniq -c`, multiset-compared old file vs new directory. Both are exit-code scripts on the lane gate, not eyeball checks.

Before ANY test-distribution slice (P2-G S7 and both core phases' test slices), one tooling commit adds a narrow test-mod-wiring ALLOW class (`#[cfg(test)]` / `#[cfg(all(test, feature = "‚Ä¶"))]` attribute lines and bare `mod <name> {`/`}` opener-closer pairs on test mods ‚Äî attribute must be attached to a mod, smuggle-proof: any non-mod line under the class is residue), with two self-test fixtures: test distribution ‚Üí PROVEN; a code line smuggled into a test-mod header ‚Üí CAUGHT. D-19's second-lander rule applies verbatim ‚Äî this script is co-edited by live Wave 1 lanes; whoever lands second merges both pattern sets and reruns the full self-test.
