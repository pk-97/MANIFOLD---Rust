# Authoring Graphs — preset JSON, from zero to a rendered look

**Status: living guide, authored 2026-07-11 (Fable) from the Scene 1 build transcript
digest + same-day authoring session. Field names verified against
`crates/manifold-core/src/effect_graph_def.rs` at time of writing.**

This is the missing manual for *authoring preset graphs as JSON* — the knowledge every
session previously re-derived from Rust source (~8 min + an error rate, measured in the
2026-07-11 Scene 1 transcript). It is not about writing primitives
([ADDING_PRIMITIVES.md](ADDING_PRIMITIVES.md)), not a node inventory
([NODE_CATALOG.md](NODE_CATALOG.md) — read it FIRST for vocabulary), and not about
organizing graphs ([GROUPING_GRAPHS.md](GROUPING_GRAPHS.md)). It is: what a preset IS
on disk, every field you will type, the edit→render→look loop, and the landmines —
each one bought with real debugging time, dated.

This guide is also the seed spec for the MCP authoring interface: each section maps to
a tool a future agent calls instead of reading this.

## 1. What a preset is

One JSON file = one `EffectGraphDef`: a typed node graph plus optional
`presetMetadata` describing its performance surface (the param cards a user sees and
binds faders/LFOs/audio to). Generators live in
`crates/manifold-renderer/assets/generator-presets/<TypeId>.json`, effects in
`assets/effect-presets/`. Same schema; generators start from `system.generator_input`
and end at `system.final_output`.

The smallest real one — `generator-presets/TrivialPassthrough.json` (canonical copy on
disk; it is load-validated by `every_bundled_preset_loads_validates_and_compiles`, so
if this quote ever drifts, trust the file):

```json
{
  "version": 1,
  "name": "TrivialPassthrough",
  "description": "D1 smoke-test generator: ...",
  "nodes": [
    { "id": 0, "typeId": "system.generator_input", "handle": "input", "nodeId": "input" },
    { "id": 1, "typeId": "node.uv_field", "handle": "uv", "nodeId": "uv" },
    { "id": 2, "typeId": "system.final_output", "handle": "final_output", "nodeId": "final_output" }
  ],
  "wires": [
    { "fromNode": 1, "fromPort": "out", "toNode": 2, "toPort": "in" }
  ]
}
```

For a full-featured worked example (groups, multi-material gltf, cards, bindings,
string params), read `generator-presets/ApricotWeather.json` end-to-end — it is the
Scene 1 preset and exercises nearly everything below.

## 2. Schema, field level

All keys are **camelCase** (`#[serde(rename_all = "camelCase")]` throughout).

**Node** (`EffectGraphNode`):
- `id: u32` — document-local wire address. Wires reference it. Not stable identity.
- `nodeId: string` — **stable identity, what bindings target.** Minted once, survives
  grouping/ungrouping/flattening. See [NODE_ID_TARGETING.md](NODE_ID_TARGETING.md).
- `typeId: string` — the primitive, e.g. `"node.bend_mesh"`, or `"group"` (§4), or a
  `system.*` boundary node.
- `handle: string?` — display/search name ONLY. Never an addressing key.
- `params: { <name>: {"type": "Float"|"Int"|"Enum"|..., "value": ...} }` — static
  param values. The envelope is typed: `{"type": "Float", "value": 0.5}`,
  `{"type": "Int", "value": -1}`, `{"type": "Enum", "value": 0}`. A param a wire or
  binding drives at runtime can still carry its default here.

**Wire** (`EffectGraphWire`): `{ "fromNode": <id>, "fromPort": "out", "toNode": <id>,
"toPort": "in" }`. Port names are the primitive's declared names — get them from
`docs/node_catalog.json` / the `primitive!` block, never guess.

**Multi-ports are name-generated by a count param.** `node.render_scene` with
`params.objects = 3` exposes `object_0..2` — each fed by one `node.scene_object` node
that itself binds mesh/transform/material/maps/instances (SCENE_OBJECT_AND_PANEL_V2_DESIGN,
shipped 2026-07-17; superseded the old parallel `mesh_k`/`material_k`/`transform_k`/…
per-object port family this section used to describe) — and `lights = N` → `light_0..N-1`
(lights kept their own ports, unaffected by that migration). Changing the count RENUMBERS
the ports — every wire into removed/shifted slots must be rewritten (bit us 2026-07-11
removing a ground plane: objects 4→3 required renumbering all three remaining objects'
wires down by one; the same renumbering now applies to `object_k` wires, one per object
instead of nine).

## 3. presetMetadata — the performance surface

Observed-complete field set (authoritative structs: `PresetMetadata` in
`crates/manifold-core/src/effects.rs`):

- `id`, `displayName`, `category`, `oscPrefix`, `available: bool`, `isLineBased`,
  `skipMode: {"kind": "never"}`.
- `params: [ParamSpec]` — the cards. Fields: `id`, `name`, `min`, `max`,
  `defaultValue`, `wholeNumbers`, `isToggle`, `isTrigger`, `formatString` ("F1"/"F2"),
  optional `isAngle: true` (renders as angle), optional `section: "Sun"` (collapsible
  header grouping on the card).
- `bindings: [Binding]` — what a card drives:
  `{"id": "swayAmount", "label": "...", "defaultValue": 0.0,
    "target": {"kind": "node", "nodeId": "sway_1", "param": "angle"}}`.
  One card id may appear in MULTIPLE bindings (fan-out: Scene 1's one Sway card drives
  `sway_1`/`sway_2`/`sway_3`). The target's `nodeId` is the node's stable `nodeId`,
  reachable inside groups without path syntax.
- `stringParams` / `stringBindings` — same pattern for strings;
  `"isFilePicker": true` gives a file-picker card (Scene 1's Model File → the three
  `gltf_mesh_source.path` params).

**Philosophy (Peter's standing rule):** cards are the performance surface. A card is
an *amount* a fader/LFO/audio can ride; the *character* of motion (gusting, easing,
phase) lives inside the graph on the beat clock. Never bake a timeline; never ship a
look whose only motion is "user wiggles the slider" either — expose amplitude, build
character in-graph.

## 4. Groups

A node with `"typeId": "group"` carries its subgraph inline:
`"group": { "interface": {"inputs": [], "outputs": [{"name": "vertices", "portType":
"Array(MeshVertex)"}, ...], "params": []}, "nodes": [...], "wires": [...],
"tint": [r,g,b,a] }`. Inner wires use inner node ids; the group's outward ports are
its `interface` (wire to them with the interface port names). `system.group_output`
inside collects the outputs. `portType` strings are advisory labels ("Texture2D",
"Material", "Scalar(F32)") — the real type check runs post-flatten. Group by
legibility per [GROUPING_GRAPHS.md](GROUPING_GRAPHS.md), not granularity.

## 5. The authoring loop

1. **Vocabulary first**: [NODE_CATALOG.md](NODE_CATALOG.md) §atoms + §presets; find
   the nearest shipped preset and read it end-to-end (§2.5 discipline). No new
   primitives without the audit.
2. **Edit the JSON.** Python surgery beats hand-editing for wire renumbering.
3. **Validate**: `cargo test -p manifold-renderer --lib preset` (~0.4s) — parse,
   port/type, and compile checks for every bundled preset.
4. **Render**: `cargo run -p manifold-renderer --release --bin render-generator-preset
   -- <PresetId> --size 1280x720 --frames 3000 --out /tmp/x.png --param id=value ...`
   — `--param` takes outer-card ids. **Always `--frames 3000`** until BUG-117
   (silent under-render while async loads converge) ships its `--wait-converged` fix.
5. **LOOK at the PNG — at an angle where failure would show.** An above-angle camera
   hides everything below a ground plane; a front angle hides depth errors. Render
   param extremes (0 / mid / max), not just defaults. Describe what you actually see;
   "wires validated" is not a look.
6. **In-app check (Peter's machine)**: the preset catalog loads from disk **once at
   app start**, so a restart is required after any preset edit. A placed clip does
   NOT embed a graph copy by default — the per-instance override is
   `Option<EffectGraphDef>` (`crates/manifold-core/src/graph_target.rs`): `None`
   means the runtime reads the catalog default, so preset edits flow to existing
   clips on restart. Only a clip whose graph was edited through the graph editor
   holds its own copy (`Some(def)`, authoritative) — those need delete + re-drop.
   (Corrected 2026-07-11 against graph_target.rs + live confirmation; the earlier
   "embeds forever" claim came from a context-rotted transcript.)

## 6. Recipes

**Multi-material glTF scan** (the ApricotWeather shape; mirrors
`node_graph::gltf_import`): one scan = one object group per glTF `material_index` —
`gltf_mesh_source(material_index=k)` + `gltf_texture_source(texture_index=k)` +
material node → `mesh_k`/`material_k`/`base_color_map_k` on `render_scene`; ONE shared
`transform_3d` wired into every group so the chunks move as a unit. `mesh_index=-1`
merged does NOT work across materials (one texture can't serve 4 UV atlases).
Photogrammetry scans may bake a calibration cube as material 0 — check per-material
bounds and skip it. Accessor `min`/`max` are LOCAL mesh space; scene-node matrices
(z-up roots, per-node placements) may not be applied by `gltf_mesh_source` — if chunk
alignment looks wrong, that's the thread to pull.

**Organic motion**: `push_along_normals` = inflate (balloon), never sway.
Base-anchored `bend_mesh` (`center` at the mesh's local base) = stems lean, tips
travel. Differential motion across a mesh must be a **continuous field** (e.g.
`mesh_ramp` weights → `bend_mesh.weights`) — per-chunk phase offsets tear the mesh at
atlas seams, because chunks are texture splits, not semantic parts. Scale sway ranges
to the asset: 0.6 rad reads as breeze on a tree, pretzel on a macro sprig.

**Camera**: default the card values to where the asset actually reads (Scene 1's scan
is macro — distance 9, not 26). `--param camDistance=...` sweeps in the render CLI
before committing defaults.

## 7. Landmines (dated, all real)

- Catalog loads at app start — restart after preset edits. Re-drop is only needed for
  clips whose graph was edited in the graph editor (those hold an override copy;
  untouched clips read the catalog). (2026-07-11, corrected same day)
- `render-generator-preset` under-renders async-loading presets silently — BUG-117.
  `--frames 3000` until fixed. (2026-07-11)
- `render_scene` fog washes out the whole frame instead of depth-grading — BUG-118,
  logged, unfixed. Leave `fog_density` 0. (2026-07-11)
- Count-param port renumbering breaks wires silently until validation. (2026-07-11)
- Verification angle: pick the camera that would SHOW the bug. Above-angle "confirmed"
  a ground fix that edge-on disproved. (2026-07-11)
- glb accessor bounds are local; Sketchfab scans carry unapplied scene-node matrices.
  (2026-07-11)
